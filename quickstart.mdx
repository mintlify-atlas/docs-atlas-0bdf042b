## Prerequisites

Before you begin, make sure you have the following:

- **Node.js 18 or later** — check with `node --version`
- **A Perplexity AI API key** — get one at [perplexity.ai](https://www.perplexity.ai/)
- **Git** — to clone the repository

<Info>
  All AI features in Resonance route through a local Express backend that proxies requests to Perplexity's API. Your API key never touches the browser.
</Info>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Resonance app repository to your local machine.

    ```bash
    git clone https://github.com/neuralboy/reasonance-app.git
    cd reasonance-app
    ```
  </Step>

  <Step title="Install dependencies">
    Install all frontend and backend dependencies in a single step.

    ```bash
    npm install
    ```

    This installs React, Vite, Express, the Perplexity-compatible OpenAI client, and all other packages defined in `package.json`.
  </Step>

  <Step title="Create your .env file">
    Create a `.env` file in the project root and add your Perplexity API key.

    ```bash
    PERPLEXITY_API_KEY=your_api_key_here
    ```

    The backend reads this value on startup. Without it, the server will start but all AI endpoints will return a `503` error.

    <Note>
      Never commit your `.env` file. The repository's `.gitignore` already excludes it.
    </Note>
  </Step>

  <Step title="Start the backend server">
    Start the Express backend on port 3001.

    ```bash
    npm run server
    ```

    You should see output similar to:

    ```
    🌟 Resonance Backend Server
    📡 Running on http://localhost:3001
    🤖 AI Status: Ready (Perplexity)
    ```

    If `AI Status` shows `Not configured`, check that your `.env` file exists and contains a valid `PERPLEXITY_API_KEY`.
  </Step>

  <Step title="Start the frontend">
    In a separate terminal, start the Vite development server.

    ```bash
    npm run dev
    ```

    The app will be available at `http://localhost:5173`.

    <Tip>
      You can run both the backend and frontend together with a single command:

      ```bash
      npm run dev:all
      ```

      This uses `concurrently` to start `npm run server` and `npm run dev` in parallel.
    </Tip>
  </Step>
</Steps>

## Verifying the installation

### Check the health endpoint

With the backend running, confirm the server is up and your API key is loaded:

```bash
curl http://localhost:3001/api/health
```

A healthy response looks like this:

```json
{
  "status": "ok",
  "message": "Resonance backend is running",
  "aiAvailable": true,
  "timestamp": "2026-03-27T12:00:00.000Z"
}
```

If `aiAvailable` is `false`, the server started without a valid API key. Re-check your `.env` file and restart the server.

### Check the status banner in the UI

Open `http://localhost:5173` in your browser. At the top of the page, you'll see a status banner:

- **Green banner** — `AI Backend connected. All features available.` Your setup is complete.
- **Amber banner** — `AI Backend not available.` The frontend cannot reach the backend. Make sure `npm run server` is running on port 3001.

## Your first action: Consciousness Mapping

Consciousness Mapping is the core feature of Resonance. It takes a raw journal entry and reflects your emotional frequency, growth edges, flow triggers, and a single next step back to you.

<Steps>
  <Step title="Navigate to Consciousness Mapping">
    Click the **Consciousness** icon (the brain) in the top navigation bar.
  </Step>

  <Step title="Write a journal entry">
    Type anything that is alive in you right now — what you're thinking about, feeling, struggling with, or excited by. There is no right format. The more honest, the more useful the reflection.

    Example:

    ```
    I've been feeling scattered lately. There's a lot of momentum at work but I'm not sure
    if it's the right direction. I keep noticing I feel most alive when I'm outside or
    talking to people I care about, but I'm spending most of my time in front of screens.
    ```
  </Step>

  <Step title="Map your frequency">
    Click **Map My Frequency**. The button will show a spinner while the backend calls the Perplexity API.

    When the analysis completes, you'll receive:

    - **Your current frequency** — a description of your present emotional-energetic state
    - **Growth edges** — areas where you're ready to expand (not broken, ready)
    - **Flow triggers** — what reliably brings you into presence
    - **Pattern observed** — a gentle observation about a recurring theme
    - **Your next step** — one actionable insight to carry into your day
  </Step>
</Steps>

<Tip>
  After reading your results, close the app and act on the next step before opening it again. That's the whole point.
</Tip>
